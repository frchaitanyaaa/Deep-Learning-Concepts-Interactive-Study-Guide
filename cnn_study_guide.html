Q2 — Max Pooling (2×2, stride 2)

Hover over any coloured region in the input to see which value "wins". Pooling shrinks the feature map while keeping the strongest activations.

Input 4×4 — 4 colour-coded regions

Bold ring = max winner in each region

Pooled output 2×2
Hover a region to inspect
4×4 → 2×2: spatial size halved, computation quartered.
Translation invariance: small shifts won't change the output.
💡 Why pool? Without it, moving an object 1 pixel would give a completely different feature map. Pooling makes the network robust to small movements — an edge at x=10 looks the same as x=11 after pooling.

💡 The elegant insight: If F(x)=0, then H(x)=x — the block becomes an identity function. Learning "do nothing" is easy. Learning the full mapping H(x) from scratch is hard. Skip connections make every extra layer optional!

💡 The pattern: R-CNN → Fast: stop repeating CNN for each region. Fast → Faster: stop using hand-crafted proposals. Faster R-CNN is fully end-to-end learnable — everything is differentiable.

Q8 — YOLO vs R-CNN trade-offs

Two fundamental approaches to object detection. Neither is universally better — pick based on your constraints.

YOLO — single pass detection

Image divided into grid. Each cell predicts boxes & classes simultaneously.

Real-time One forward pass Full image context
Weaknesses: struggles with very small or densely packed objects. Grid cells can only predict a fixed number of boxes.
R-CNN family — propose then classify
1. Extract feature map (one CNN pass)
2. Generate region proposals (RPN)
3. ROI Pooling → classify each region
4. Refine bounding boxes
More accurate Slower Better for small objects
CriterionYOLOFaster R-CNN
Speed✅ 30–100+ FPS❌ 5–10 FPS
Accuracy (mAP)⚠️ Good✅ Better
Small objects⚠️ Harder✅ Better

💡 Rule of thumb: Live video, drones, autonomous cars → YOLO. Medical imaging, satellite analysis, forensics → Faster R-CNN.

Why YOLO wins for real-time
• Single forward pass — no separate proposal stage
• Full image context — fewer background false positives
• Compact — smaller variants run on edge devices
• 30–100+ FPS on a modern GPU
Real-world deployments
🚗 Autonomous driving — 60 FPS needed
📹 Surveillance — continuous 24/7 streams
🤖 Robotics — low-latency reactions
🏭 Factory QC — real-time defect detection
